# セキュリティルール

## アクセス禁止ファイル

- `.env` / `.env.*` → 読み書き禁止
- `*.pem` / `*secret*` → 読み書き禁止
- `.github/workflows/**` → 編集禁止

## 危険コマンド禁止

- `rm -rf` → 禁止
- `git push --force` → 禁止
- `git reset --hard` → ユーザー確認後のみ

## サーバーファイル編集前のバックアップ

- SSH経由でサーバーのファイルを編集する前に必ず `.bak` コピーを作成する
- 例: `cp target.php target.php.bak`
- DBの変更（記事の一括操作等）は大きな作業前のみ `wp db export` でダンプする

## コード品質

- **XSS**: ユーザー入力は必ずサニタイズ・エスケープしてから出力する
- **SQLインジェクション**: プリペアドステートメント必須、文字列結合でのクエリ組み立て禁止
- **認証不備**: 認証チェックをバイパスする実装を作らない
- **シークレット**: API キー・パスワードをコードにハードコードしない（環境変数 / wp-config.php で管理）
- **CSRF**: フォーム・Ajax リクエストには必ず nonce を付与して検証する
- **オープンリダイレクト**: リダイレクト先 URL はホワイトリストで制限する
