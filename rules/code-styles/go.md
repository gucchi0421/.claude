# Go コーディングスタイル

## フォーマット

- `gofmt` / `goimports` を必ず通す
- エクスポートする識別子には godoc コメントを付ける

## エラーハンドリング

- エラーを握りつぶさない（`_` で捨てない）
- `fmt.Errorf("context: %w", err)` でラップして文脈を付ける
- 判定は `errors.Is` / `errors.As` を使う

## 設計

- インターフェースは小さく保つ（1〜3 メソッド）
- goroutine を起動したら必ず終了を制御する（`context.Context` / `chan struct{}`）
- `panic` は本当に回復不能な場合のみ
- 関数型アプローチを優先し、グローバル変数による状態共有を避ける

## コメント

- WHY が非自明な場合のみ記述する
- エクスポート関数は一行サマリーを付ける（godoc 準拠）
