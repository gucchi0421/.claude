# TypeScript コーディングスタイル

## 型

- `any` は禁止。`unknown` を使い、型ガードで絞る
- 型アサーション（`as`）は最終手段。型ガードを優先する
- 関数の引数・戻り値は型を明示する
- `interface` よりも `type` を優先（extends が必要な場合は `interface`）
- nullable は `Type | null`（`?` は関数引数の省略可能のみ）

## 関数・設計

- 関数型アプローチを優先し、副作用を最小化する
- 純粋関数を基本とし、破壊的な操作（`push`, `splice` 等）を避ける
- `async/await` を使い、`.then()` チェーンは避ける

## エラーハンドリング

- エラーを握りつぶさない
- `catch (e)` の `e` は `unknown` として扱い、`instanceof Error` 等で絞る
- 意味のあるメッセージを付けて再スローまたはログ出力する

## コメント

- WHY が非自明な場合のみ記述する（WHAT はコードが語る）
- 型定義に対する長い JSDoc は不要
