# HTML コーディングスタイル

## セマンティクス

- セマンティックなタグを使う（`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` 等）
- `<div>` / `<span>` は意味を持たない要素。役割が明確な場合は適切なタグを選ぶ
- 見出しは階層順に使う（h1 → h2 → h3、スキップしない）

## 必須属性

- `<html>` に `lang="ja"` を必ず付ける
- `<img>` に `alt` 属性を必ず付ける（装飾目的のみの画像は `alt=""`）
- `<a>` に `href` のない使い方は避ける

## アクセシビリティ

- フォーム要素には `<label>` を紐付ける（`for` 属性または `aria-label`）
- キーボードで全インタラクティブ要素に到達できること
- コントラスト比は WCAG AA 基準（4.5:1 以上）を満たすこと

## パフォーマンス

- 重要な CSS は `<head>` で読み込む
- スクリプトは `defer` または `async` を付ける（ただしプロジェクトの読み込み方式に従う）
- `loading="lazy"` をスクロール外の画像に付ける
