# サムネイル生成ルール

## スクリプト

`~/.claude/scripts/generate_thumbnail.py`

Pollinations.ai（model=flux, enhance=true）でサムネイルを生成し、`(pwd)/.claude/articles/[slug].jpg` に保存する。
WP へのアップロードは **wp-operator エージェントが担当**する。

## 基本構文

```bash
python3 ~/.claude/scripts/generate_thumbnail.py \
  --title "記事タイトル" \
  --slug {slug} \
  --scene "（英語シーン描写・50語以上）" \
  --style ghibli
```

保存先は自動的に `$(pwd)/.claude/articles/{slug}.jpg` になる。

## オプション一覧

| オプション | 必須 | 説明 |
|---|---|---|
| `--title` | ✅ | 記事タイトル（WPメディアタイトルにも使用） |
| `--slug` | 推奨 | 記事スラッグ。省略時は `/tmp/thumbnail.jpg` に保存 |
| `--scene` | 推奨 | 英語シーン描写。省略するとtitleそのままが使われ品質が落ちる |
| `--style` | 任意 | スタイルプリセット（後述）。デフォルト: `ghibli` |
| `--output` | 任意 | 保存先パスを明示指定（`--slug` より優先） |
| `--seed` | 任意 | 乱数シード（デフォルト: 42）。変えると別パターンが生成できる |

## スタイルプリセット

| `--style` | タッチ |
|---|---|
| `ghibli`（デフォルト） | スタジオジブリ風・柔らかい水彩イラスト・暖かいトーン |
| `ink_watercolor` | インク線画＋水彩ウォッシュ・エディトリアル調・アンバートーン |
| `flat_vector` | フラットベクターイラスト・パステルカラー・モダングラフィック |
| `ukiyo_e` | 浮世絵・木版画風・伝統的日本画 |

## プロンプト品質基準（必須）

### やること
- `--scene` に**英語50語以上**で記述する
- 具体的な物体・色・光源・雰囲気・質感を列挙する
- 記事の内容・ターゲット読者・サービスの雰囲気を反映させる

### やらないこと
- `photorealistic` / `realistic` / `photograph` などフォトリアル系ワードを使わない
- `people` / `person` / `man` / `woman` / `child` など人物描写を入れない（自動で `no people` が付くが、入れると生成AIが無視するため）
- 短い抽象的な描写（例: `festival scene`）だけで済まさない
- カテゴリ・ジャンルで固定プロンプトを使い回さない（記事ごとに個別作成）

## 改善経緯（なぜこの設計になったか）

以前の設計はカテゴリマップでジャンル→プロンプトを変換していたが、以下の問題が発生した：

1. カテゴリに対応するキーがない記事はすべて同じ画像（都市風景）になった
2. `photorealistic` 指定で人物が恐ろしい見た目になった
3. 短いプロンプトでAIが補完した結果、サイトイメージと無関係な画像になった

→ `--scene` で記事ごとに個別の長文英語プロンプトを渡す設計に変更。スタイルはプリセットで再現性を確保。

→ WPアップロードは wp-operator に分離し、スクリプトは画像生成・保存のみに責務を絞った。
