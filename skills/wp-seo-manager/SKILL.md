# WordPress SEO 自動化スキル

ディレクターが指揮し、SEO エンジニア・ライター・コーダーが専門作業を分担する。
ユーザーは Chatwork で承認するだけ。詳細は各エージェント指示書（agents/）を参照。

# 対策ワード

「ホームページ制作　大阪」
「ホームページ作成　大阪」
「WEB 制作　大阪」

## サイト情報

- URL: https://your-site.com
- 対策 KW: （対象 KW を記入）
- ログイン情報: secrets.md 参照

## エージェント構成

| エージェント   | 指示書                 | 担当                                               |
| -------------- | ---------------------- | -------------------------------------------------- |
| ディレクター   | agents/director.md     | 全体指揮・承認管理・Chatwork・スケジュール・Notion |
| SEO エンジニア | agents/seo-engineer.md | 調査・競合分析・計画・AIOSEO 設定・効果検証        |
| ライター       | agents/writer.md       | 本文品質チェック・リライト・追記・FAQ 生成         |
| コーダー       | agents/coder.md        | テーマファイル修正・ソースコード変更（diff 必須）  |

エージェント間の引継ぎは agents/context-template.md を使う。

## フロー（固定順）

```
ディレクター → SEOエンジニア（audit + research + plan）
             ↓
ディレクター → Excelレポート送付 → Chatwork承認待ち
             ↓ Go
ディレクター → バックアップ実施（backup.md）
             ↓
ディレクター → SEOエンジニア（AIOSEO設定変更）
             ├→ ライター（本文追記・リライト・FAQ）※並列可
             └→ コーダー（テーマ修正 ※diff承認が別途必要）
             ↓ 全完了
ディレクター → Notion記録 → Chatwork完了通知 → 効果検証スケジュール設定
```

## 鉄則

- 実行は承認後のみ
- 実行前に必ず WPvividBackup でバックアップを取得し、完了を確認してから作業開始
- **ソースコードを変更する際は必ず diff を提示してユーザーの承認を得てから実行する**
- 変更は diff で明示（変更前 → 変更後）、変更対象（固定ページ/AIOSEO/設定）を明記
- **タイトル・ディスクリプションの変更前に必ず管理場所を特定すること**
  - AIOSEO が出力しているなら → AIOSEO 管理画面で変更
  - テーマの header.php や functions.php にべた書きなら → ソースコード編集（diff 提示・承認必須）
  - WP 標準設定なら → WP 管理画面「設定 → 一般」または投稿編集画面で変更
  - 管理場所が不明なまま変更してはならない
- E-E-A-T 意識
- タイトル 32 文字以内・ディスクリプション 120 文字前後
- 実行完了後は必ず Notion に記録 → Chatwork に通知
- plan フェーズでは「提案」ではなく「そのまま使えるコンテンツ＋変更対象マップ」を生成する

## スケジュール設定（自発的な定期監視）

以下の 2 種類の自動実行を /schedule スキルで設定する。

### 毎週月曜の定期監視

毎週月曜の 10 時に audit フェーズを自動実行し、異常があれば Chatwork に報告する。
（稼働時間: 平日 10 時〜19 時のみ。それ以外の時間帯にはスケジュールを入れない）

報告内容:

- 対策 KW 順位の前週比変化（急落があれば「要対応」フラグ）
- 404 エラーの増減
- AIOSEO スコアが大きく下がった記事
- AIO 引用状況の変化
- 異常なし → 「今週は異常なし」と 1 行で報告

通常の audit→research→plan→ 実行フローは起動しない。異常検知の監視のみ。
要対応の項目が見つかった場合は「対応が必要です。承認いただければ対策を進めます」と Chatwork に提案する。

### 施策後の効果検証（4 週間後）

全エージェントの作業完了後、ディレクターが /schedule で「4 週間後に効果検証を実行」を設定する。
検証内容は references/report.md § 3 を参照。

## 顧客追加時の Notion メンテ

新顧客の SEO 対策を開始する際:

1. Notion DB の「顧客名」Select に新しい顧客ドメインを追加
2. SKILL.md の「サイト情報」に顧客情報を追記
3. secrets.md に顧客の WP・GSC のログイン情報を追記
