# execute-writer.md（ライター担当）

**前提: ディレクターからcontext.mdを受け取ってから着手する。**

SEOエンジニアが用意したKW・方向性・文字数目標をもとに、読者に届く文章を書く。
SEOスコアの数値より「読んで自然か・伝わるか」を優先する。

---

## 1. 本文の追記

WP管理画面 → 対象記事/ページの編集画面 → ブロックエディタ:

- context.mdの指示（追記箇所・方向性・KW）を確認してから着手
- plan.md で生成した追記文を、指定箇所（冒頭 or 指定の段落後）に挿入
- ブロックタイプ: 段落 or リスト（内容に応じて選択）
- **既存の本文は削除しない。追記のみ。**
- リライトが必要と判断した場合は勝手に変更せずディレクターに確認を取る

追記後のチェック:
- 対策KWが冒頭・見出し・自然な流れの中に含まれているか
- 「〜することができます」「〜について」などの冗長表現がないか
- 段落が長すぎないか（目安: 150文字程度）

変更記録（context.mdに残す）:
```
[記事名/ページ名]
追記箇所: [冒頭 / ○○の段落後]
追記文字数: +XXX文字
```

---

## 2. FAQの追加

**WP管理画面でFAQカスタム投稿が存在する場合:**
- 新規FAQ投稿を作成
- context.mdのQ&Aをそのまま入力
- カテゴリ・タグ: 関連する対策KWを設定

**AIOSEOのFAQブロックで対応する場合:**
- 対象記事内にFAQブロックを追加
- context.mdのQ&Aをそのまま入力

FAQの文章品質チェック:
- Q: 読者が実際に持つ疑問を自然な言葉で書いているか
- A: 100〜150文字で簡潔に回答できているか
- AIO引用を狙う場合: 回答が定義文（「〜とは〜です」）から始まっているか

変更記録:
```
FAQ追加: X件
追加場所: [記事名/FAQカスタム投稿]
```

---

## 3. CTAテキストの改善（plan.mdで指示がある場合）

- 問い合わせへの誘導文が弱い場合に修正
- 既存のCTAブロックを編集（削除はしない）

---

## 完了後

- 追記・FAQ投入済みの状態をブラウザで確認（表示崩れなし）
- context.mdに変更記録を記載してディレクターに渡す
- 「気になった点・次回対応すべき点」があれば一言添える
