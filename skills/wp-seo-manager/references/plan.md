# references/plan.md

## 本日の SEO 対策計画（YYYY/MM/DD）

research.md の結果をもとに、**具体的なコンテンツ・コード**を計画段階で生成してユーザーに提示する。
「〜してください」で終わらず、実際に使えるものを出す。

---

### 優先A（スコア低 + 更新古い + 競合との差が大きい）

各記事について以下を全て生成して提示:

#### 【記事名】

**メタ情報の修正案**
```
タイトル: 「[具体的なタイトル案]」（XX文字）
ディスクリプション: 「[具体的なディスクリプション案]」（XX文字）
フォーカスKW: [KW]
```

**本文の追記案（そのまま使える文章）**

競合が持っていて自社にない要素を補完する文章を生成:
- 定義文（AIO引用対策）
- 実績・数値を含む説明文
- 対応エリア・特徴の箇条書き
- CTAテキスト改善案

**追加するFAQ（Q&A形式、そのまま使える）**

```
Q: [質問文]

Q: [質問文]
```

### 提示方法（Excelレポート → Chatwork送付 → 承認待ち）

計画生成後はチャットに貼らず、以下の手順で Chatwork に送付して承認を待つ。

#### 1. Excelレポートを生成

このスキルのディレクトリ（SKILL.mdと同じ場所）から実行:

```bash
python scripts/generate_report.py
```

生成されるファイル: OSの一時ディレクトリ（Linux/Mac: `/tmp/`、Windows: `%TEMP%\`）に `seo_report_YYYYMMDD.xlsx` として出力される。

シート構成:
| シート名 | 内容 |
|---------|------|
| 調査結果 | audit + research のサマリー（順位・スコア・競合ギャップ） |
| 作業計画 | 優先A/B/Cの変更対象・変更内容一覧 |
| 変更Diff | 変更箇所ごとの「変更前 → 変更後」差分表 |

#### 2. Chatworkにファイル添付で送付

Playwright でブラウザを使い以下の手順で送付:

1. `https://www.chatwork.com/#!rid337490434` を開く
2. ファイル添付ボタン（クリップアイコン）をクリック
3. `browser_file_upload` で `/tmp/seo_report_YYYYMMDD.xlsx` を選択
4. 本文に以下を入力して送信:

```
[SEO対策計画] YYYY/MM/DD

Excelファイルをご確認ください。
━━━━━━━━━━━━
■ 対象ページ数: X件
■ 優先A（即対応）: X件
■ 優先B（機会対応）: X件
■ 優先C（中長期）: X件

ご確認後、承認の場合は「Go」、差し戻しの場合は「NG: [理由]」とご返信ください。
```

#### 3. 承認待ちポーリング

Playwright で Chatwork を監視し、ユーザーの返信を待つ。

```
監視間隔: 2分ごとにスクリーンショット取得
検出キーワード:
  承認 → 「Go」「go」「OK」「承認」「実行して」
  差し戻し → 「NG」「ng」「否」「待って」「修正」
タイムアウト: 2時間（超えたら停止してユーザーに確認）
```

- 「Go」を検出 → そのまま references/backup.md へ進む
- 「NG」を検出 → 差し戻し内容を確認して計画を修正し再送
- タイムアウト → 作業を停止してユーザーに声がけ
