# スケジュールエージェントサマリースキル

## トリガー

- スケジュール実行されたエージェントが作業完了時に呼び出す
- `/schedule-agent-summary` とユーザーが入力したとき（手動確認用）

## タスクログファイルの特定方法

環境変数 `SCHEDULE_LOG_FILE` が設定されていればそのパスを使う。
設定されていない場合は `.claude/logs/schedule/` 内の最新ファイルを使う。

## 出力先

### タスクログ（追記）
`{SCHEDULE_LOG_FILE}` — 1タスク1ファイル。関わった全エージェントが順番に追記する。

### インデックス
`{プロジェクトルート}/.claude/logs/schedule/index.md` — 全エージェントの最終状態を上書き更新。

## タスクログへの追記フォーマット

```markdown
## {agent-name}

- {実施内容を箇条書き（ファイル名・WP投稿ID・変更内容を含む）}
- 結果: 成功 ✅ / 失敗 ❌ / 部分完了 ⚠
- {公開URLなど定量情報}

**次回への引き継ぎ**: {次回このエージェントがすべきこと}
```

## インデックスの書き方

`index.md` は各エージェントの最終状態を上書き更新する（追記ではない）。

```markdown
# スケジュールエージェント実行インデックス

最終更新: YYYY-MM-DD HH:MM

| エージェント | 最終実行 | 最後の作業 | 次にすべきこと |
|---|---|---|---|
| writer | 2026-05-28 14:00 | GA4記事公開（ID:7900） | retina記事リライト |
| seo-engineer | 2026-05-27 09:00 | トップページmeta更新 | カテゴリページ対応 |
| wp-operator | 2026-05-28 10:00 | 下書き3本リライト公開 | 残り下書きリスト確認 |
```

## 手順

1. `SCHEDULE_LOG_FILE` 環境変数またはscheduleディレクトリの最新ファイルでタスクログを特定する
2. 上記フォーマットで自分のエージェントセクションをタスクログ末尾に追記する
3. `index.md` を読み込み、自分のエージェント行を更新して上書き保存する

## 実行後

「ログを `{ファイルパス}` に追記したのだ！インデックスも更新したのだ！」と一言報告する。
