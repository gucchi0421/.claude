# スケジュールエージェントサマリースキル

## トリガー

- スケジュール実行されたエージェントが作業完了時に呼び出す
- `/schedule-agent-summary` とユーザーが入力したとき（手動確認用）

## 出力先

### 個別ログ
`{プロジェクトルート}/.claude/logs/schedule/YYYYMMDD_{agent-name}.md`

- `{agent-name}` は実際に作業したエージェント名（`writer` / `seo-engineer` / `wp-operator` など）
- 同日・同エージェントのファイルが存在する場合は末尾に `---` を挟んで追記

### インデックス
`{プロジェクトルート}/.claude/logs/schedule/index.md`

- 全エージェントの最終実行状態を1ファイルで管理
- 次回起動時にエージェントが読んで「前回何をしたか」を把握する

## 個別ログの書き方

```markdown
## HH:MM — {一行で何をしたか}

### 実行タスク
- タスク種別: 新規記事執筆 / SEO改善 / リライト / etc.
- 対象: キーワード・記事ID・ページ名など

### 実施内容
- 具体的にやったこと（ファイル名・WP投稿ID・変更内容を含む）

### 結果
- 成功 ✅ / 失敗 ❌ / 部分完了 ⚠
- 公開URL・アイキャッチID・スコアなど定量情報を残す

### 次回への引き継ぎ
- このエージェントが次回実行時にすべきこと
- ハマったこと・注意点
```

## インデックスの書き方

`index.md` は各エージェントの最終状態を上書き更新する（追記ではない）。

```markdown
# スケジュールエージェント実行インデックス

最終更新: YYYY-MM-DD HH:MM

| エージェント | 最終実行 | 最後の作業 | 次にすべきこと |
|---|---|---|---|
| writer | 2026-05-28 14:00 | GA4記事公開（ID:7900） | retina記事リライト |
| seo-engineer | 2026-05-27 09:00 | トップページmeta更新 | カテゴリページ対応 |
| wp-operator | 2026-05-28 10:00 | 下書き3本リライト公開 | 残り下書きリスト確認 |
```

## 手順

1. プロジェクトルートを特定する（`.claude/` を持つ最も近い祖先ディレクトリ）
2. `LOG_DIR={プロジェクトルート}/.claude/logs/schedule` を作成（なければ）
3. `LOG_FILE=$LOG_DIR/$(date +%Y%m%d)_{agent-name}.md` を決定
4. 上記フォーマットで個別ログを生成・追記する（既存なら `---` 挟んで追記）
5. `index.md` を読み込み、該当エージェントの行を更新して上書き保存する

## ファイル先頭（個別ログ新規作成時のみ）

```markdown
# スケジュール実行ログ YYYY-MM-DD — {agent-name}

```

## 実行後

「ログを `{ファイルパス}` に書いたのだ！インデックスも更新したのだ！」と一言報告する。
