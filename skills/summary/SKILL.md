# サマリースキル

## トリガー

- ユーザーが `/summary` と入力したとき
- 会話が長くなってきたときにぼくから提案する（目安: 20ターン超 or `/compact` を促すタイミング）

## 出力先

`{プロジェクトルート}/.claude/logs/summary/YYYYMMDD.md`

- 同日ファイルが存在する場合は**末尾に追記**（ファイル量産しない）
- プロジェクトルートは `.claude/` ディレクトリを持つ最も近い祖先ディレクトリ

## サマリーの書き方

会話コンテキスト全体を読んで、以下の構造でMarkdownを書く。
**Gemini・外部APIは使わない。ぼく自身が会話を読んで書く。**

```markdown
## セッション HH:MM — {一行で何をしたセッションか}

### ユーザーの指示
- 箇条書きで「何をしてほしかったか」を記載

### 実施した作業
- 何をどうしたか（ファイル名・変更内容を含む）

### なぜそうしたか
- 判断理由・背景・制約・トレードオフ

### 留意事項・気づき
- 次回のぼくへの引き継ぎ事項
- ハマったこと・注意点
- ユーザーから修正された行動があれば必ず記録する

### 残課題
- 未完了・次回やること（なければ省略）
```

## 手順

1. プロジェクトルートを特定する
2. `SUMMARY_DIR={プロジェクトルート}/.claude/logs/summary` を作成（なければ）
3. `SUMMARY_FILE=$SUMMARY_DIR/$(date +%Y%m%d).md` を決定
4. 上記フォーマットでサマリーを生成する
5. ファイルが存在する場合は末尾に `---` セパレータを挟んで追記、存在しない場合は新規作成

## ファイル先頭（新規作成時のみ）

```markdown
# 作業サマリー YYYY-MM-DD — {プロジェクト名}

```

## 実行後

「サマリーを `{ファイルパス}` に書いたのだ！」と一言報告する。
